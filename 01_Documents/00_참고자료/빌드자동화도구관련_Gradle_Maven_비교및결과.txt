※ Maven vs Gradle 차이

# 빌드 관리 도구
   빌드 자동화 도구라고 보면 됨
   프로젝트 생성, 테스트 빌드, 배포 등의 작업을 위한 전용 프로그램
   개발 시 필요한 라이브러리들을 설정 파일을 통해 자동으로 다운로드 해주고, 이를 간편히 관리해주는 도구

# Maven
  Java용 프로젝트 관리도구, Apache의 Art 대안으로 만드렁짐
  prom.xml 파일 : 빌드 중인 프로젝트, 빌드 순서, 각종 라이브러리 종속성 관계가 평시됨
  외부 저장소에서 필요한 라이브러리와 플러그인들을 다운로드 한 후, 로컬 시스템의 캐시에 저장하는 방식
  
  간단하게 prom.xml 파일에 필요한 환경을 기록하면 해당 파일을 통해 알아서 환경을 구축해주는 것으로 보면 될듯함

# Gradle
   Apache Maven 과 Apache Art 에서 볼 수 있는 개념들의 대안으로 나온 것
   오픈소스 툴, Maven보다 간결하다고함
   안드로이드 프로젝트의 빌드 시스템
   자세한 방식이 안나와있음

# 비교
   ㅇ 빌드에 접근하는 방식
       - Gradle : 작업 의존성 그래프를 기반으로 함
       - Maven : 고정적이고 선형적인 단계의 모델을 기반으로 함
    ㅇ 성능면
        - Gradle의 빌드 시간이 더 짧음
        - 커스터마이징이 간단함
        - concurrent(동시 제어였나) 아무튼 동시로 가능
    ㅇ 결론
         - Maven보다 Gradle 이 더 최근에 나옴 => 당연히 더 좋을듯
         - 속도, 캐시 사용 안전성 등 Gradle이 더 좋아보임
